# Comment Reply
**Status: LOCKED (2026-06-19).** A video (≈15–30s) that answers a real (or representative) comment/question/objection on screen. Feels native, not like an ad — you're "just replying." Excellent for handling objections and for riding engagement on a post that's getting traction. Read `00-engine.md` first.

## Why it works
The on-screen comment *is* the hook — it borrows curiosity and social proof, and it pre-frames the video as a helpful reply rather than a pitch. Answering an objection out loud disarms it for everyone watching with the same doubt.

## Sourcing the comment
1. **Ask the member for a real comment first.** Best case is an actual comment the brand received (paste it in). Real always beats invented.
2. **If they don't have one, suggest 2–3 candidates and let them pick.** Build each from a **purchase-blocking objection** — the questions a prospect is most likely to have that would stop them buying (start from brand-data.md §9 Objections, but filter to "what actually kills the sale"). Present them as options, member chooses or edits.
3. **Representative comments are allowed but must stay honest.** Frame them as "a question we get a lot" / "someone always asks…" — **never attribute a fabricated quote to a specific named person** or invent a screenshot of a real account.

## Hook logic (0–3s)
Show the comment as a sticker/screenshot and **read it aloud** with a reaction. Pick a comment that's a real objection, a sceptical take, or a "does it actually…?" question (pull from brand-data.md §9 Objections). The spicier/more sceptical the comment, the better the stop. Generate reaction lines via the hook engine with VIDEO TYPE = "Comment Reply".

## Beats
| Beat | ~Time | What to do |
|---|---|---|
| 1 — The comment | 3s | On-screen comment + you reading it / reacting ("Okay this is a fair question…" / "Someone said…"). |
| 2 — Validate it | 4s | Agree it's a real concern — don't get defensive. "Honestly, I thought the same." Builds trust. |
| 3 — The answer | 8–12s | Answer with a specific + proof. Show, don't tell — demonstrate the thing the comment doubted. |
| 4 — The turn | 4s | The reframe: why the doubt is exactly why the product exists / why it's better than they feared. |
| 5 — The ask | 3s | Light CTA: "so yeah — if you were wondering that too, [action]." |

**One objection, done well** — that's the default. You may add **one quick bonus point only if it directly reinforces the same objection** (e.g. doubt = "tastes weak" → main answer demonstrates strength, bonus adds "and one bottle is 15 cups so you're not rationing it"). Don't pivot to an unrelated second objection — that splits the video.

## Worked example sketch (generic — replace)
Comment on screen: "Looks like it'd taste watered down tbh." → "Fair — I assumed that too." → [demonstrates] "It's actually concentrated, so one [dose] = [result]. Watch." → "That's the whole reason it works." → "If that was your worry, link's below."
