# UGC Scripts — Brand Config (fallback)

> This file is the **fallback** source of brand inputs for the UGC script types. If the brand-intelligence skill's `brand-data.md` exists, the skill uses that instead and ignores this file. Fill this in only if you're running UGC Scripts without a brand profile.
>
> Keep it short — these are the minimum inputs the seven video types need. Don't invent product facts; leave blanks and the skill will ask.

## The five hook inputs (used by every type)
- **Brand name:** [BRAND NAME]
- **Hero product (one line on what it does):** [PRODUCT — what it does]
- **Founder's name + 1-line personality:** [NAME — vibe] (only needed for Founder/Interview types)
- **The category frustration the brand exists to solve:** [SPECIFIC FRUSTRATION]
- **Target audience (one line):** [WHO BUYS IT]

## Voice
- **Sounds like:** [3 words — e.g., warm, direct, a bit cheeky]
- **Never says:** [corporate phrases / banned words]
- **Talks like:** [who — e.g., "a knowledgeable mate", not a brand]

## Proof & specifics to draw on
- **Best proof points:** [rating/review count, results, guarantee, "X sold"]
- **Real customer phrases (verbatim):** ["…", "…"]
- **The transformation (before → after):** [from X to Y]
- **Top objections to pre-empt:** [objection → quick answer]

## Product facts the scripts can state safely
- **Claims we can make:** [substantiated claims]
- **Claims to avoid:** [unverified claims]
- **Price / offer (only if you want it in-script):** [price / offer]

## Action / CTA
- **Default ask:** [what you want the viewer to do — e.g., "tap the link, try it"]
- **Where the video sends them:** [product page / profile / link]
