# TEACH — what good looks like

The framework for turning a repeatable task into a reusable skill. Capture each letter with the member, then build. This file shows what a strong answer looks like for each, with examples.

> Worked example used throughout: a member who, every week, writes a "back in stock" email when a sold-out product returns.

## T — Task: the one repeatable job
The single, repeatable thing the skill does. Not a project, not three jobs — one.
- **Ask:** "What's the one task you do over and over that you'd hand to Claude?"
- **Good:** "Write a back-in-stock email when a product comes back into stock."
- **Too big:** "Run all our email marketing." (That's many skills — pick one.)
- **Test:** could you do this same task next week, and the week after? If yes, it's a skill.

## E — Explain: when it should fire (the trigger) + the outcome
Two things: **when** Claude should reach for this skill, and **what a good result looks like**. The trigger becomes the skill's description — the most important line, because it decides whether the skill ever runs.
- **Ask:** "When should this kick in — what words or situations? And what does a great finished result look like?"
- **Good trigger:** "Whenever I say 'back in stock email', name a product that's returned, or paste a restock notification." Be generous and a little pushy with the phrases so it doesn't under-fire.
- **Good outcome:** "A short, on-brand email with a subject line, a 'it's back' hook, the product, and a buy button — ready to drop into Klaviyo."
- **Test:** if you described the trigger to a stranger, would they know when to use it?

## A — Actions: the steps, briefed like day one
How the task gets done, step by step, as if training someone on their first day. Order matters; note any "if this, then that."
- **Ask:** "Walk me through exactly how you do it now — every step."
- **Good:** "1) Confirm which product and its price/link. 2) Pull 2–3 selling points from the product page. 3) Write a subject line + preview. 4) Write the email in our voice (warm, short). 5) Add the buy button. 6) Flag if the product's been out >30 days so we mention 'by popular demand'."
- **Keep it concrete and numbered.** Vague steps ("make it good") produce vague skills.

## C — Connect: the tools, templates, and examples
What the skill needs to do the job well — data sources, tools, brand files, and examples of a great result.
- **Ask:** "Does it need any tools or data (Shopify, Klaviyo, a sheet, the site)? Any templates, brand guidelines, or a past example that nailed it?"
- **Good:** "Pull product details from Shopify; match our brand voice file; follow the structure of last month's best back-in-stock email (attached)."
- Examples are gold — one great past result teaches the skill more than a page of instructions.

## H — Hone: test on real work, then tune
How you'll check it and make it better. A skill is rarely perfect on run one.
- **Ask:** "What's a real example we can test it on? How will we know it worked, and what should it always double-check?"
- **Good:** "Test it on the [product] that just restocked. It worked if I'd send it with light edits. It should always check the price is current and the link works before finishing."
- Build a **self-check** into the skill from the H answers (the things it must verify before it's done).

---

## Writing the description (the trigger) — the part that matters most
When you generate the skill, the `description` carries the Explain answers. It must say **what the skill does AND when to use it**, packed with the member's real trigger phrases. Skills tend to *under*-trigger, so lean slightly pushy.

- **Weak:** "Writes back-in-stock emails."
- **Strong:** "Write an on-brand back-in-stock email for a returned product. Use whenever the member says 'back in stock', 'restock email', names a product that's come back, or pastes a restock alert — even if they don't say the word 'email'. Pulls product details and matches brand voice."

## One job per skill
If the member lists several tasks, build the first cleanly and note the rest as future skills. A skill that tries to do everything fires at the wrong time and does each thing worse.
